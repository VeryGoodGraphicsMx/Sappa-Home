La lógica en GitHub + Netlify normalmente es esta: index.html = HOME principal del sitio cualquier otro archivo .html = página secundaria ejemplo: sappa-catalogo.html about.html maintenance.html Entonces si quieres poner una página temporal tipo: “Estamos trabajando para ti” botón de WhatsApp tienes dos opciones: Opción 1 — Reemplazar temporalmente el HOME (la más fácil) Simplemente: creas un nuevo index.html subes commit a GitHub Netlify redeploy automático ahora ESA página será el home No tienes que “decirle a GitHub” cuál es home. Siempre busca automáticamente: index.html Eso es lo que estás usando actualmente en tu sitio La estructura quedaría así: /index.html ← maintenance temporal /sappa-catalogo.html /assets/ y luego cuando quieras regresar: vuelves a subir tu index original. Opción 2 — Mejor práctica profesional Guardar tu home original y hacer switch rápido. Ejemplo: /index-original.html /index.html ← maintenance /sappa-catalogo.html Así nunca pierdes el original.
